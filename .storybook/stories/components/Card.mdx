# Card

O componente `Card` é um contêiner versátil que agrupa conteúdo relacionado e ações em um único elemento, fornecendo uma base visual consistente para diversos tipos de informação.

```jsx

```

## Uso Básico

```jsx
import React from 'react';
import { Card, Typography, Button } from '@porto/js-library-corp-hubv-porto-ocean';

function Example() {
  return (
    <Card>
      <Card.Header>
        <Typography variant="subtitle1">Título do Card</Typography>
      </Card.Header>
      
      <Card.Body>
        <Typography variant="body1">
          Este é um exemplo básico de um card com cabeçalho, corpo e rodapé.
          O conteúdo principal fica no corpo do card.
        </Typography>
      </Card.Body>
      
      <Card.Footer>
        <Button variant="primary">Ação Principal</Button>
        <Button variant="secondary">Ação Secundária</Button>
      </Card.Footer>
    </Card>
  );
}
```

## Props

### Card

| Prop | Tipo | Padrão | Descrição |
|------|------|--------|-----------|
| `children` | `ReactNode` | - | Conteúdo do card |
| `variant` | `'default' \| 'outlined' \| 'elevated'` | `'default'` | Variante visual do card |
| `hoverable` | `boolean` | `false` | Se verdadeiro, adiciona efeito de hover ao card |
| `onClick` | `(e: MouseEvent) => void` | - | Função chamada quando o card é clicado |
| `className` | `string` | - | Classe CSS adicional para o componente |
| `style` | `CSSProperties` | - | Estilos inline adicionais |

### Card.Header

| Prop | Tipo | Padrão | Descrição |
|------|------|--------|-----------|
| `children` | `ReactNode` | - | Conteúdo do cabeçalho |
| `className` | `string` | - | Classe CSS adicional para o cabeçalho |
| `style` | `CSSProperties` | - | Estilos inline adicionais |

### Card.Body

### Card.Footer

| Prop | Tipo | Padrão | Descrição |
|------|------|--------|-----------|

## Acessibilidade

O componente `Card` segue as práticas recomendadas de acessibilidade:

- Utiliza estrutura semântica apropriada para o conteúdo
- Para cards clicáveis, implementa atributos ARIA adequados e suporte a navegação por teclado
- Mantém contraste adequado entre o conteúdo e o fundo do card
- Garante que imagens incluam texto alternativo descritivo
- Preserva a ordem de leitura lógica para leitores de tela

## Boas Práticas

- Use cards para agrupar informações relacionadas e criar uma hierarquia visual clara
- Mantenha o conteúdo dos cards conciso e focado em um único tópico ou ação
- Seja consistente no uso de cabeçalhos, imagens e ações entre cards similares
- Para cards clicáveis, forneça indicações visuais claras (como efeito de hover)
- Considere a responsividade ao definir o layout de múltiplos cards
- Use as subcomponentes `Card.Header`, `Card.Body` e `Card.Footer` para manter uma estrutura consistente
- Evite sobrecarregar cards com muitas ações ou informações
