# IconButton

O componente `IconButton` é uma variação especializada do botão projetada especificamente para exibir ícones, ocupando menos espaço na interface e mantendo a interatividade.



## Importação

```jsx
import { IconButton } from '@porto/js-library-corp-hubv-porto-ocean';
```

## Uso Básico

```jsx
import React from 'react';
import { IconButton } from '@porto/js-library-corp-hubv-porto-ocean';
import { Trash } from 'phosphor-react';

function Example() {
  return (
    <IconButton
      icon={<Trash size={20} />}
      aria-label="Excluir item"
      onClick={() => console.log('Item excluído')}
    />
  );
}
```

## Props

| Prop | Tipo | Padrão | Descrição |
|------|------|--------|-----------|
| `icon` | `ReactNode` | - | Ícone a ser exibido no botão |
| `variant` | `'primary' \| 'secondary' \| 'ghost'` | `'primary'` | Variante visual do botão |
| `color` | `'default' \| 'insurance' \| 'banking' \| 'health' \| 'negative'` | `'default'` | Esquema de cores do botão |
| `size` | `'small' \| 'medium' \| 'large'` | `'medium'` | Tamanho do botão |
| `disabled` | `boolean` | `false` | Define se o botão está desabilitado |
| `rounded` | `boolean` | `true` | Se o botão deve ter formato circular |
| `aria-label` | `string` | - | Descrição acessível da ação do botão (obrigatório para acessibilidade) |
| `className` | `string` | - | Classe CSS adicional para o componente |
| `style` | `CSSProperties` | - | Estilos inline adicionais |
| `onClick` | `(event: MouseEvent<HTMLButtonElement>) => void` | - | Função chamada quando o botão é clicado |
| `...rest` | `ButtonHTMLAttributes<HTMLButtonElement>` | - | Outras props nativas do elemento button |

## Acessibilidade

O componente `IconButton` segue as práticas recomendadas de acessibilidade:

- **Sempre forneça um `aria-label`**: Como o botão não contém texto visível, é essencial fornecer uma descrição acessível da ação que o botão executa.
- **Estado desabilitado**: Quando o botão está desabilitado, o atributo `disabled` é aplicado corretamente, informando aos leitores de tela sobre o estado do botão.
- **Foco visível**: O botão mostra um indicador de foco visível quando navegado por teclado.
- **Suporte a teclado**: O botão pode ser acionado usando a tecla Enter ou Space, seguindo os padrões de acessibilidade.

## Boas Práticas

- **Sempre use `aria-label`**: Como o IconButton não contém texto, é crucial fornecer um rótulo acessível.
- **Escolha ícones reconhecíveis**: Use ícones que sejam facilmente reconhecíveis e representem claramente a ação que executam.
- **Mantenha consistência**: Use o mesmo ícone para a mesma ação em toda a aplicação.
- **Considere tooltips**: Para ações menos óbvias, considere adicionar um tooltip que aparece ao passar o mouse sobre o botão.
- **Tamanho adequado**: Certifique-se de que o botão seja grande o suficiente para ser facilmente clicável em dispositivos de toque (pelo menos 44x44 pixels para a área de toque).
- **Use variantes apropriadas**: Use a variante "ghost" para ações menos proeminentes e "primary" para ações principais.
- **Agrupe botões relacionados**: Quando houver vários IconButtons relacionados, agrupe-os visualmente em uma barra de ferramentas ou painel.
- **Evite sobrecarga**: Não use muitos IconButtons próximos uns dos outros para evitar confusão visual.
