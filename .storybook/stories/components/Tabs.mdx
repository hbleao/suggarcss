# Tabs

O componente `Tabs` permite organizar conteúdo em múltiplas abas, exibindo apenas uma seção de cada vez, economizando espaço e melhorando a organização da interface.



## Importação

```jsx
import { Tabs } from '@porto/js-library-corp-hubv-porto-ocean';
```

## Uso Básico

```jsx
import React, { useState } from 'react';
import { Tabs, Typography } from '@porto/js-library-corp-hubv-porto-ocean';

function Example() {
  const [activeTab, setActiveTab] = useState('1');
  
  const handleChange = (key) => {
    setActiveTab(key);
  };
  
  return (
    <Tabs activeKey={activeTab} onChange={handleChange}>
      <Tabs.Tab key="1" title="Visão Geral">
        <Typography variant="body1">
          Esta é a aba de visão geral do produto. Aqui você encontrará informações básicas e uma introdução ao produto.
        </Typography>
      </Tabs.Tab>
      
      <Tabs.Tab key="2" title="Especificações">
        <Typography variant="body1">
          Esta aba contém todas as especificações técnicas detalhadas do produto.
        </Typography>
      </Tabs.Tab>
      
      <Tabs.Tab key="3" title="Avaliações">
        <Typography variant="body1">
          Aqui você pode ver as avaliações e comentários dos usuários sobre o produto.
        </Typography>
      </Tabs.Tab>
    </Tabs>
  );
}
```

## Props

### Tabs

| Prop | Tipo | Padrão | Descrição |
|------|------|--------|-----------|
| `children` | `ReactNode` | - | Componentes Tabs.Tab |
| `activeKey` | `string` | - | Chave da aba ativa |
| `defaultActiveKey` | `string` | - | Chave da aba ativa inicial (quando não controlado) |
| `onChange` | `(key: string) => void` | - | Função chamada quando a aba ativa muda |
| `variant` | `'default' \| 'card' \| 'line'` | `'default'` | Variante visual das abas |
| `size` | `'small' \| 'medium' \| 'large'` | `'medium'` | Tamanho das abas |
| `tabPosition` | `'top' \| 'bottom' \| 'left' \| 'right'` | `'top'` | Posição das abas em relação ao conteúdo |
| `centered` | `boolean` | `false` | Se verdadeiro, centraliza as abas |
| `className` | `string` | - | Classe CSS adicional para o componente |
| `style` | `CSSProperties` | - | Estilos inline adicionais |

### Tabs.Tab

| Prop | Tipo | Padrão | Descrição |
|------|------|--------|-----------|
| `key` | `string` | - | Identificador único da aba |
| `title` | `ReactNode` | - | Título da aba |
| `children` | `ReactNode` | - | Conteúdo da aba |
| `disabled` | `boolean` | `false` | Se verdadeiro, desativa a aba |
| `icon` | `ReactNode` | - | Ícone a ser exibido antes do título da aba |
| `className` | `string` | - | Classe CSS adicional para a aba |

## Acessibilidade

O componente `Tabs` segue as práticas recomendadas de acessibilidade:

- Implementa o padrão WAI-ARIA para tabs, usando `role="tablist"`, `role="tab"` e `role="tabpanel"`
- Suporta navegação completa por teclado (Tab para focar nas abas, setas para navegar entre abas)
- Associa corretamente cada aba ao seu painel de conteúdo usando `aria-controls` e `aria-labelledby`
- Indica o estado atual das abas com `aria-selected`
- Fornece feedback visual claro para os estados de foco, seleção e desabilitado

## Boas Práticas

- Use títulos concisos e descritivos para as abas
- Limite o número de abas a um máximo de 5-7 para evitar sobrecarga visual
- Organize o conteúdo de forma lógica, agrupando informações relacionadas em uma mesma aba
- Evite aninhamento excessivo de componentes Tabs
- Considere usar ícones junto com texto para melhorar a compreensão visual
- Mantenha a consistência no estilo e comportamento das abas em toda a aplicação
- Para conteúdo extenso, considere adicionar rolagem vertical dentro de cada painel
- Evite usar tabs para etapas sequenciais (use stepper em vez disso)
