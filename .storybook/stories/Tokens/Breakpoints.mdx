# Breakpoints

O Ocean Design System utiliza um conjunto de breakpoints para criar layouts responsivos que se adaptam a diferentes tamanhos de tela.

## O que são tokens de breakpoints?

Tokens de breakpoints são pontos de interrupção predefinidos que representam larguras específicas de viewport onde o layout da interface deve mudar para se adaptar melhor ao tamanho da tela. Eles são fundamentais para criar designs responsivos.

## Breakpoints do Ocean

## Larguras de Container

Junto com os breakpoints, o Ocean também define larguras máximas para containers em cada breakpoint:

| Breakpoint | Largura do Container | Token |
|------------|----------------------|-------|
| XS (320px) | 304px | $grid-container-xs |
| SM (768px) | 680px | $grid-container-sm |
| MD (1024px) | 896px | $grid-container-md |
| LG (1224px) | 1224px | $grid-container-lg |
| XL (1600px) | 1600px | $grid-container-xl |

## Uso dos Breakpoints

Os breakpoints devem ser usados para criar layouts responsivos que se adaptem a diferentes tamanhos de tela.

### Como usar os breakpoints em SCSS

```scss
.my-component {
  // Estilo base (mobile-first)
  padding: $space-sm;

  // Tablet portrait e acima
  @media (min-width: $breakpoint-sm) {
    padding: $space-md;
  }

  // Desktop e acima
  @media (min-width: $breakpoint-lg) {
    padding: $space-lg;
  }
}
```

### Como usar os breakpoints em CSS

```css
.my-component {
  /* Estilo base (mobile-first) */
  padding: var(--ocean-space-sm);
}

@media (min-width: 768px) {
  .my-component {
    padding: var(--ocean-space-md);
  }
}

@media (min-width: 1224px) {
  .my-component {
    padding: var(--ocean-space-lg);
  }
}
```

## Boas Práticas

- Adote uma abordagem mobile-first, começando com estilos para telas menores e adicionando media queries para telas maiores
- Evite criar breakpoints personalizados; use os breakpoints padrão do sistema sempre que possível
- Teste seu design em vários tamanhos de tela, não apenas nos breakpoints exatos
- Considere não apenas a largura, mas também a altura da tela em alguns casos
- Use containers com largura máxima para garantir que o conteúdo permaneça legível em telas muito largas
- Adapte não apenas o layout, mas também o tamanho da tipografia, espaçamento e outros elementos para cada breakpoint
