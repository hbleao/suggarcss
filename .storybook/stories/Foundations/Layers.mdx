# Camadas (Z-Index)

O Ocean Design System utiliza um sistema de camadas (z-index) para controlar a sobreposição de elementos na interface, garantindo uma hierarquia visual consistente.

## O que são tokens de camadas?

Tokens de camadas (z-index) são variáveis predefinidas que representam a ordem de empilhamento dos elementos na interface. Eles determinam quais elementos aparecem na frente ou atrás de outros quando há sobreposição.

## Escala de Camadas do Ocean

O Ocean define uma escala progressiva de camadas que vai de elementos base até modais e elementos que devem sempre aparecer no topo:

## Uso dos Tokens de Camadas

Os tokens de camadas devem ser usados para controlar a ordem de empilhamento dos elementos na interface.

### Aplicação Comum

- **$z-index-base (10)**: Elementos base da interface, como cabeçalhos, rodapés, cards
- **$z-index-menu (20)**: Menus de navegação, dropdowns
- **$z-index-overlay-tooltip (30)**: Overlay para tooltips
- **$z-index-tooltip (40)**: Tooltips, popovers
- **$z-index-always-on-top (50)**: Elementos que devem sempre aparecer acima de outros, como notificações
- **$z-index-overlay-modal (60)**: Overlay escuro para modais
- **$z-index-modal (70)**: Modais, diálogos

### Como usar os tokens de camadas

Em SCSS:
```scss
.header {
  z-index: $z-index-base;
}

.dropdown-menu {
  z-index: $z-index-menu;
}

.modal-overlay {
  z-index: $z-index-overlay-modal;
}

.modal {
  z-index: $z-index-modal;
}
```

Em CSS:
```css
.header {
  z-index: var(--ocean-z-index-base);
}

.dropdown-menu {
  z-index: var(--ocean-z-index-menu);
}

.modal-overlay {
  z-index: var(--ocean-z-index-overlay-modal);
}

.modal {
  z-index: var(--ocean-z-index-modal);
}
```

## Boas Práticas

- Use sempre os tokens de camadas predefinidos em vez de valores arbitrários
- Mantenha a consistência usando os mesmos tokens para elementos com funções similares
- Evite criar novos valores de z-index; use os tokens existentes sempre que possível
- Lembre-se que o z-index só funciona em elementos posicionados (position: relative, absolute, fixed ou sticky)
- Considere o contexto de empilhamento: elementos com z-index só competem dentro do mesmo contexto
- Evite "guerras de z-index" mantendo uma hierarquia clara e consistente
